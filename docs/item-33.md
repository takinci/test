# Item 33 — Testing on external data

### Describe the external data used to evaluate the completed algorithm. If no external testing is performed, note and justify this limitation. If there are differences in structure of annotations or data between the training set and the external testing set, explain the differences, and describe the approach taken to accommodate the differences.

Explanation and Elaboration: Machine learning models typically demonstrate poorer performance on external data–data drawn from different sources than their training data–due to limited ability to generalize. Generalizability is crucial in medical imaging AI, as the data seen in clinical deployments is rarely drawn from exactly the same distribution as the training dataset. Absence of external testing is a severe limitation; in most cases, work is not publishable without external testing. External datasets are ideally as different as is reasonably practical from the training data; the sources and demographics of the external data should be described, so readers can assess the extent of the difference. Ideally the structure and definitions of annotations on the external data are concordant with those on the internal data, but that may not be the case when a previously created dataset is used for external testing. When such unavoidable differences exist, the manuscript should detail the approach employed to mitigate their impact.
